# 资源、美术与特效规范

## 分辨率与资源规格建议

- 游戏基准分辨率：`1920x1080`

建议资源尺寸：

- 角色：`256x256`
- 小怪：`128x128` 或 `256x256`
- Boss：`512x512`
- 小图标 / 子弹：`64x64` 或 `128x128`
- 普通特效：`256x256`
- 大招特效：`512x512` 到 `1024x1024`
- UI 图标：`64 / 128 / 256`
- 按钮：`320x120` 或 `420x140`
- 面板：`512x256`、`1024x512`
- Build 卡：`420x220`

## 资源格式建议

- 角色 / 特效 / UI：优先 `PNG`
- 背景大图：`JPG` 或 `PNG`
- 文字尽量与底图分离，便于后续修改文案和本地化

## 特效设计总原则

- 特效必须服务于玩法辨识，而不是只追求好看
- 玩家应当一眼看出：
  这是谁的技能
  这招打的是哪里
  这招和上一种效果有什么不同

## 特效与判定关系

- 特效范围尽量贴近真实伤害范围
- 不允许“看起来打到了但没伤害”
- 不允许“没碰到却判定中了”
- 如果两者冲突，优先调整判定去贴特效，而不是让玩家猜

## 剑士普通攻击当前实现约定

当前剑士普通攻击使用：

- 特效场景：
  `res://effects/sword/slash3/slasheffect3.tscn`
- 方向逻辑：
  与鼠标方向相关，但不是简单顺着鼠标方向播放
- 主判定：
  当前是隐藏矩形判定
- 要求：
  特效大小变化时，伤害范围也要一起跟

## 剑士普通攻击方向逻辑摘要

完整说明见桌面：
历史草稿文件 `特效方向逻辑.txt`（未纳入当前仓库；后续应迁移为仓库内文档后再引用）

当前共识摘要：

- 鼠标负责决定攻击出现在角色哪一侧
- 斩击轴线与“角色到鼠标的连线”垂直
- 取朝屏幕下方的那一侧
- 鼠标在右半边时额外做水平镜像
- 特效旋转中心必须按可见内容中心校正
- 特效与主判定必须共用同一套空间逻辑

## 特效替换原则

- 后续替换角色特效时，优先保持逻辑接口不变
- 尽量替换资源，不重写整套判定逻辑
- 只有在新特效表达方式完全变了时，才调整对应的判定逻辑

## 当前美术阶段定位

- 当前大量素材仍处于“测试贴图 / 测试特效”阶段
- 现阶段目标不是统一最终画风
- 而是先保证：
  可替换
  可运行
  可快速验证观感
