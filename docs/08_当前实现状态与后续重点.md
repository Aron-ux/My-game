# 当前实现状态与后续重点

## 当前已经基本具备的内容

- 主菜单
- 存档位选择
- 主线准备界面功能版
- 战斗主循环
- 三角色切换
- 普攻 / 进场技 / 退场技 / 大招
- 经验与升级
- Build 菜单
- 普通怪 / 精英 / Boss 基础框架
- HUD / 暂停菜单 / 游戏结束界面
- BGM 与继续游戏
- 开发者模式

## 当前已验证的主爽点

- 切人节奏成立
- 大招演出能形成明显高潮
- 三个角色已有基本差异
- 项目已经脱离“只有概念没有手感”的阶段

## 当前主要短板

### 构筑深度仍然不够

- 构筑比起传统幸存者 Like 还不够支撑长局游玩
- 很多 Build 仍然偏向数值而不是战斗方式变化

### 构筑与角色特征仍需继续绑定

- 三个角色必须继续强化各自的独特演出语言
- 剑士应该更像夸张斩击体系
- 枪手应该更像夸张弹幕体系
- 术师应该更像轰炸和区域控制体系

### 工程复杂度已经明显上升

- `player.gd` 和 `enemy.gd` 已经很大
- 继续堆功能会降低后续维护效率

## 当前适合优先推进的方向

1. 把已经讨论好的构筑方案继续实装
2. 继续加强构筑带来的视觉差异
3. 完善主线准备界面和故事模式骨架
4. 继续丰富怪物生态和 Boss 剧本
5. 逐步补齐第 4、第 5 个角色的占位设计

## 当前不建议立刻做的事情

- 大规模重构全部代码
- 一口气做完整剧情演出系统
- 一开始就做最终美术成品规格
- 把所有未验证系统同时并行推进

## 推荐的下一阶段开发顺序

1. 稳住当前战斗可玩版本
2. 继续打磨 Build 深度与角色差异
3. 把主线准备和关卡推进流程跑通
4. 继续扩展关卡内容和难度模式
5. 再进入更系统的剧情、营地互动和长期养成

## 结论

- 项目已经不是“做不出来”的阶段
- 当前核心任务不是从零开始想，而是把现有系统持续整理、扩展、去同质化
- 这套文档的目的，就是让后续继续开发时不需要反复从聊天记录里捞结论
