## 目录结构

- `assets/`
  存放菜单背景、角色贴图等资源
- `effects/`
  存放特效场景与特效贴图，例如剑士斩击 `slasheffect3`
- `scenes/`
  存放主要游戏场景、UI 场景、战斗单位场景
- `scripts/`
  存放核心逻辑脚本
- `shaders/`
  存放材质与 shader
- `ui/`
  预留 UI 资源目录

## 主要脚本职责

- `scripts/main.gd`
  战斗主控。负责关卡时间、刷怪、精英/Boss 触发、UI 装配、继续游戏、自动存档。

- `scripts/player.gd`
  绝对核心脚本。负责玩家移动、三角色状态、普攻、切人、进场技、退场技、大招、Build、HUD 数据、特效触发、角色样式等。

- `scripts/enemy.gd`
  敌人基础行为、成长、精英/Boss 模式切换、Boss 技能逻辑。

- `scripts/enemy_bullet.gd`
  敌方弹幕行为。

- `scripts/bullet.gd`
  我方子弹行为，主要服务枪手相关攻击。

- `scripts/hud.gd`
  负责时间、等级、HP、MP、经验条、Boss 血条、队伍信息等显示。

- `scripts/level_up_ui.gd`
  负责升级菜单、Build 卡展示、多级菜单交互。

- `scripts/pause_menu.gd`
  ESC 菜单逻辑，包括继续、重开、返回主菜单、音量设置。

- `scripts/main_menu.gd`
  主菜单逻辑，负责开始、继续、设置、开发者模式入口、菜单 BGM。

- `scripts/save_manager.gd`
  存档系统。负责故事存档、战斗存档、继续游戏目标、元数据。

- `scripts/story_data.gd`
  主线数据定义。包括关卡表、角色池、风格样式、默认故事档结构。

- `scripts/story_prep.gd`
  主线准备界面。负责当前编队、角色顺序、风格切换、进入下一关。

- `scripts/developer_mode.gd`
  开发者模式开关，控制无敌、无限大招、全卡候选、禁用存档等。

- `scripts/bgm_player.gd`
  BGM 播放器逻辑，负责音量配置、静音、暂停、续播位置。

## 当前架构特点

- 项目逻辑高度集中在 `player.gd` 和 `main.gd`
- 优点：单人开发推进快，很多内容可以直接实装
- 风险：后续继续膨胀时，角色逻辑、Build 逻辑、特效逻辑会越来越难维护

## 后续建议

- 不急着大重构
- 但后续可逐步拆分：
  `player.gd` -> 角色战斗 / Build / 特效 / 队伍轮转
  `enemy.gd` -> 普通敌人 / 精英敌人 / Boss 行为
