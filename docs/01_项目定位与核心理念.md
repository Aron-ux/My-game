# 项目定位与核心理念

## 项目定位

- 类型：Godot 2D 幸存者 Like + 三角色切换战斗
- 团队条件：单人开发，优先做出“可玩、可测、可迭代”的版本
- 当前开发目标：先把战斗主循环、角色爽点、主线框架和菜单骨架跑通，再逐步加深内容

## 核心体验

- 队伍由 3 名角色组成，始终是 1 人站场、其余角色后台协同
- 玩家通过 `WASD` 走位、鼠标定向、`Q/E` 切人、`R` 释放大招形成节奏
- 核心节奏不是站桩自动清屏，而是：
  `走位普攻 -> 切换角色入场 -> 释放大招`

## 设计优先级

- 演出反馈 > 操作爽感 > 构筑深度 > 数值平衡
- 视觉表现优先于纯数字堆叠
- 如果玩家“感受不到收益”，优先加强演出而不是先拉数值

## 角色设计方向

- 每个角色都能当主C，不做纯辅助
- 剑士：近身压制、生存、吸血/减伤、夸张斩击演出
- 枪手：远程 poke、稳定点杀、满屏弹幕演出
- 术师：范围轰炸、区域控制、慢节奏高覆盖演出

## 构筑设计方向

- 保留三栏结构：战斗栏 / 连携栏 / 技能栏
- 构筑不只追求数值成长，而要能改变视觉画面、战斗方式、角色高光
- 鼓励三名角色都值得切、都不残废、都能有高光，但高光方式必须不同

## 大招原则

- 大招必须“贵且强”，不能廉价连发
- 价值排序应当明显：
  普攻收益 < 切人收益 < 大招收益
- 玩家应该更多靠普攻和切人维持节奏，大招是珍贵资源与高潮按钮

## 项目当前阶段结论

- 已经验证可玩的爽点：切人节奏、大招演出、角色切换带来的轮转感
- 仍需继续打磨的核心：构筑的广度、深度、辨识度，以及更强的视觉反馈
