# 脚本职责树与关键数据流

## 一、脚本职责树

### 1. 战斗主控层

- `main.gd`
  负责战斗主场景的总控
  主要职责：
  - 找到玩家节点
  - 装配 HUD / 升级菜单 / 暂停菜单 / 失败界面
  - 控制生存时间、关卡进度、刷怪、精英、Boss
  - 处理继续游戏、自动存档、主线关卡上下文
  - 在战斗中持续刷新 HUD

### 2. 玩家战斗层

- `player.gd`
  当前项目最核心脚本
  主要职责：
  - 玩家移动与面朝方向
  - 三角色切换与队伍顺序
  - 普攻逻辑
  - 进场技 / 退场技 / 后台协同
  - 大招释放与后续收益
  - 经验、升级、等级成长
  - Build 候选池、Build 结果应用
  - 属性训练
  - 角色风格样式
  - 与 HUD 的数据输出
  - 大量战斗特效和命中逻辑

### 3. 敌人与弹幕层

- `enemy.gd`
  负责普通怪、精英、Boss 的基础行为
  主要职责：
  - 敌人移动与追击
  - 远程怪、加速怪、冲刺怪等行为模式
  - 状态异常：减速、易伤、流血
  - Boss 阶段切换与招式逻辑
  - 敌人被击败后的掉落与回报

- `enemy_bullet.gd`
  负责敌方弹幕实体逻辑

- `bullet.gd`
  负责我方子弹实体逻辑，主要服务枪手与部分技能

### 4. UI 层

- `hud.gd`
  负责战斗信息 UI
  主要职责：
  - 时间、等级、经验
  - HP / MP / 符印
  - 队伍面板
  - 切人 CD / 接力提示
  - Boss 血条

- `level_up_ui.gd`
  负责升级 / Build 菜单
  主要职责：
  - 一级 / 二级 / 三级菜单
  - 长按与悬停展开
  - 最终效果卡进度展示
  - 术语说明

- `pause_menu.gd`
  负责 ESC 菜单

- `game_over_ui.gd`
  负责失败结算与重开入口

### 5. 菜单与主线流程层

- `main_menu.gd`
  负责主菜单逻辑

- `save_select.gd`
  负责存档位选择

- `story_prep.gd`
  负责主线准备界面 / 功能版营地

- `story_data.gd`
  负责主线关卡表、角色池、风格样式等静态数据

### 6. 基础服务层

- `save_manager.gd`
  负责战斗存档、故事存档、继续游戏目标

- `bgm_player.gd`
  负责 BGM、音量配置、静音、续播位置

- `developer_mode.gd`
  负责开发者模式总开关

## 二、关键数据流

### 1. 启动流程

`project.godot`
-> `main_menu.tscn`
-> `main_menu.gd`
-> 选择：
- 主线模式 -> `save_select.tscn`
- 继续游戏 -> 上次继续目标场景
- 开发者模式 -> `main.tscn`

### 2. 主线流程

`save_select.gd`
-> 选择存档位
-> `save_manager.gd` 建立或读取故事档
-> `story_prep.tscn`
-> `story_prep.gd`
-> 进入下一关
-> `main.tscn`
-> `main.gd`

### 3. 战斗内核心数据流

`main.gd`
-> 初始化 `player`
-> 读取故事关卡上下文
-> 连接玩家信号
-> 定时刷新 HUD

`player.gd`
-> 战斗中触发：
- `experience_changed`
- `level_up_requested`
- `stats_changed`
- `health_changed`
- `mana_changed`
- `died`

`main.gd`
接收这些信号后：
- 更新 HUD
- 打开升级菜单
- 处理死亡与结算

### 4. 升级 / Build 流程

`player.gd`
-> 判断获得经验后是否升级
-> 生成 Build 候选
-> 发出 `level_up_requested`

`main.gd`
-> 打开 `level_up_ui`

`level_up_ui.gd`
-> 玩家选择卡片
-> 发出 `upgrade_selected`

`main.gd`
-> 把结果回传给 `player.gd`

`player.gd`
-> 应用属性成长 / Build 成长
-> 更新角色战斗状态

### 5. 存档与继续流程

`main.gd`
-> 每隔一段时间自动调用 `save_manager.gd`
-> 保存当前战斗状态

退出或切回菜单后：
- `main_menu.gd` 查询 `save_manager.gd`
- 如果有继续目标，显示“继续游戏”

继续游戏时：
- 回到 `main.tscn`
- `main.gd` 尝试恢复运行状态
- BGM 按记录位置附近恢复

## 三、当前工程里的实际“核心枢纽”

当前最关键的 3 个枢纽脚本：

1. `player.gd`
   真正的战斗核心

2. `main.gd`
   战斗流程与系统总控

3. `save_manager.gd`
   主线 / 继续游戏 / 存档位的连接枢纽

## 四、继续开发时最该注意的地方

- 新功能尽量先判断应该挂在：
  `main.gd`
  还是 `player.gd`
  还是单独服务脚本
- 不要继续把“纯 UI 逻辑”塞进 `player.gd`
- 不要把“纯数据定义”继续硬写进大量 `match` 分支里
